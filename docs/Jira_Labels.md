## Recommended Label Rules

- Use area labels to identify ownership, such as `backend` or `frontend`.
- Use feature labels to identify product capability, such as `dashboard` or `kanban`.
- Use quality labels when the work is primarily non-functional, such as `performance` or `security`.
- Use `bug` only for defects against accepted behavior.
- Use `enhancement` for improvements to existing behavior.
- Use `release` only for deployment and production readiness work.
